Qu'est-ce que Calendar++ ?

Calendar++ est une bibliothèque C++ complète et sophistiquée conçue pour manipuler, convertir et afficher les dates dans les calendriers du monde entier. Elle prend en charge 16 calendriers majeurs : du grégorien standard aux systèmes traditionnels comme le chinois, le hébraïque, l'islamique, le maya et bien d'autres.

L'objectif de cette bibliothèque est de fournir une API simple et intuitive permettant aux développeurs d'intégrer facilement la gestion multi-calendriers dans leurs applications C++. Que vous développiez une application de planification, un outil historique, un logiciel de gestion ou tout autre projet nécessitant des dates, Calendar++ vous offre une solution complète, robuste et facile à utiliser.

La bibliothèque repose sur des algorithmes éprouvés et une architecture modulaire qui garantit la précision des conversions et la fiabilité des calculs. Tous les calendriers sont implémentés selon leurs règles traditionnelles, assurant ainsi une exactitude absolue pour toutes les opérations.

Installer Calendar++

Téléchargement et intégration

Calendar++ est une bibliothèque header-only : un seul fichier calendar.h contient toute la logique. Cette approche présente de nombreux avantages : pas de compilation séparée, pas de gestion de dépendances complexes, et une intégration immédiate dans votre projet. Il vous suffit d'inclure ce fichier dans votre code source pour bénéficier de toutes les fonctionnalités.

La bibliothèque est entièrement autonome et ne nécessite aucune installation particulière. Elle est conçue pour fonctionner sur tous les compilateurs C++17 et plus récents, que vous soyez sur Windows (MSVC, MinGW), Linux (GCC, Clang) ou macOS (Clang). La compatibilité est assurée grâce à l'utilisation de fonctionnalités standard du langage, sans recours à des extensions propriétaires.

Pour l'utiliser, il vous suffit de placer le fichier calendar.h dans le répertoire de votre projet, puis d'ajouter la directive d'inclusion dans vos fichiers sources. La bibliothèque est prête à l'emploi immédiatement après inclusion, sans aucune configuration supplémentaire.

Étapes d'installation

1

Téléchargez calendar.h depuis le bouton ci-dessus

2

Copiez le fichier dans le dossier source de votre projet

3

Ajoutez #include "calendar.h" dans votre code

4

Compilez avec g++ -std=c++17 main.cpp -o program

Exemple de compilation

g++ -std=c++17 mon_programme.cpp -o mon_programme ./mon_programme

Le flag -std=c++17 est obligatoire car la bibliothèque utilise des fonctionnalités du standard C++17 (comme les structured bindings et les initialisations de classes).

Classe Date — Le cœur de la bibliothèque

La classe Date est l'élément central de Calendar++. Elle représente une date dans un calendrier spécifique avec ses composantes : jour, mois, année et le type de calendrier associé. Cette classe est conçue pour être immutable par défaut (les opérations de modification créent de nouvelles instances) et offre une interface fluide pour toutes les opérations calendaires.

Toutes les dates sont stockées sous forme d'un nombre de jours juliens (JDN) qui sert de référence universelle pour les conversions et les calculs. Ce mécanisme interne garantit des conversions exactes entre tous les calendriers supportés, quel que soit le système d'origine. Le JDN est calculé automatiquement lors de la création d'une date et est mis à jour lors des opérations arithmétiques.

La classe Date offre une API complète couvrant toutes les opérations nécessaires : création de dates, conversion entre calendriers, opérations arithmétiques (addition de jours, mois, années), validation, normalisation, formatage, et extraction d'informations (jour de semaine, numéro de semaine, jour de l'année, etc.). Elle prend également en charge la localisation pour les noms de mois et de jours.

// Exemples d'utilisation de la classe Date Date greg(25, 12, 2024); // 25 décembre 2024 (grégorien) Date islamic(1, 1, 1446, CalendarType::ISLAMIC); // 1er muharram 1446 Date converted = greg.convertTo(CalendarType::ISLAMIC); Date future = greg.addDays(30); // Ajoute 30 jours string formatted = greg.toString("DD/MM/YYYY"); // "25/12/2024"

Classe Calendar — Affichage et représentation

La classe Calendar est dédiée à l'affichage des calendriers mensuels, annuels ou pluriannuels. Elle permet de générer des représentations textuelles des calendriers dans différents formats et langues, que ce soit pour la console, l'impression ou l'intégration dans des interfaces utilisateur.

Cette classe prend en charge la localisation complète : vous pouvez afficher les noms des mois et des jours dans 6 langues différentes (français, anglais, arabe, hébreu, chinois, persan). Elle gère également le premier jour de la semaine (dimanche ou lundi selon les cultures) et offre une mise en page claire et lisible avec des colonnes alignées et des séparateurs.

La classe Calendar peut afficher des calendriers dans n'importe quel type de calendrier supporté par la bibliothèque. Elle s'appuie sur la classe Date pour les calculs sous-jacents et offre des méthodes pour afficher des mois individuels, des années complètes ou des plages d'années. Elle fournit également des informations détaillées sur chaque calendrier.

// Exemples d'utilisation de la classe Calendar Calendar cal(GREGORIAN, 1, "fr"); // Calendrier grégorien, semaine débutant lundi, français cal.printYear(2025); // Affiche l'année 2025 complète cal.printMonth(2025, 6); // Affiche juin 2025 cal.printCalendarInfo(); // Affiche les informations du calendrier

Classification des calendriers

Calendar++ couvre les 4 grandes familles de calendriers : solaires, lunaires, luni-solaires et historiques. Chaque calendrier a été implémenté avec soin pour respecter les règles traditionnelles et les conventions culturelles de chaque système.

Les calendriers solaires (grégorien, julien, éthiopien, copte, persan, etc.) sont basés sur la révolution de la Terre autour du Soleil. Ils ont une année d'environ 365,2422 jours, avec des années bissextiles pour compenser le décalage. Le grégorien est le calendrier civil le plus utilisé dans le monde, tandis que le persan est le calendrier officiel de l'Iran.

Les calendriers lunaires (islamique) suivent les phases de la Lune, avec des mois de 29 ou 30 jours et une année de 354 ou 355 jours. Le calendrier islamique est utilisé pour les fêtes religieuses et le calcul de la zakat.

Les calendriers luni-solaires (hébraïque, chinois) combinent les deux approches : des mois lunaires et des années solaires ajustées par des mois intercalaires. Le calendrier hébraïque est utilisé pour les fêtes juives, et le calendrier chinois pour les traditions asiatiques.

Les calendriers historiques (maya, républicain français, bahá'í, sikh) représentent des systèmes culturels ou religieux spécifiques, avec des structures uniques comme le compte long maya ou les 19 mois du calendrier bahá'í.

Conversion de dates — Le cœur de Calendar++

La conversion de dates est le processus de transformation d'une date d'un système calendaire à un autre. Calendar++ assure des conversions précises et fiables via la méthode convertTo(). Cette opération est fondamentale car elle permet de travailler avec des dates provenant de différents calendriers et de les comparer ou les combiner.

Le mécanisme de conversion repose sur le nombre de jours juliens (JDN), un compteur universel de jours qui sert d'intermédiaire entre les calendriers. Ce système a été développé par l'astronome Joseph Scaliger au XVIe siècle et est aujourd'hui la référence standard pour les conversions entre calendriers. Le JDN est un entier qui compte les jours écoulés depuis le 1er janvier 4713 av. J.-C. (calendrier julien).

L'utilisation du JDN garantit que les conversions sont bijectives (chaque date correspond à un JDN unique) et reproductibles (la même date donnera toujours le même JDN). Ce mécanisme est totalement transparent pour le développeur : il suffit d'appeler convertTo() sur une date pour obtenir la date correspondante dans un autre calendrier.

Par exemple, pour convertir une date grégorienne en date islamique, Calendar++ calcule d'abord le JDN de la date grégorienne en utilisant la formule de conversion, puis utilise cet entier pour reconstruire la date dans le calendrier islamique en appliquant l'algorithme inverse. Ce processus est instantané et extrêmement précis, sans perte d'information.

Opérations arithmétiques — Manipuler les dates

Calendar++ permet des opérations arithmétiques sur les dates : ajout ou soustraction de jours, mois, années avec les méthodes addDays(), addMonths() et addYears(). Ces opérations sont essentielles pour la planification, le calcul de délais, la génération de séries de dates, et bien d'autres applications.

Les opérations arithmétiques sont conçues pour être intelligentes et respecter les règles de chaque calendrier. Par exemple, ajouter un mois à une date du 31 janvier donnera le 28 ou 29 février selon l'année, et non une date invalide. De même, l'ajout d'années respecte les années bissextiles et les particularités des calendriers lunaires ou luni-solaires.

La bibliothèque gère également les dates limites et les débordements : si vous ajoutez un nombre important de jours, les mois et les années seront ajustés automatiquement. Les opérations sont réalisées en utilisant le JDN comme référence, garantissant des résultats cohérents quel que soit le calendrier de la date d'origine.

En plus des opérations arithmétiques, Calendar++ offre la méthode daysBetween() pour calculer la différence en jours entre deux dates. Cette méthode est utile pour calculer des durées, des âges ou des écarts temporels.

Multilingue — Localisation avancée

La bibliothèque supporte 6 langues : français, anglais, arabe, hébreu, chinois et persan. Les noms des mois et des jours sont disponibles dans toutes ces langues, ainsi que les formats d'affichage des dates. Cette fonctionnalité est cruciale pour les applications internationales et les logiciels destinés à des publics multiculturels.

La localisation est gérée de manière élégante et extensible : il suffit de passer le code de langue ("fr", "en", "ar", "he", "zh", "fa") aux méthodes appropriées pour obtenir l'affichage souhaité. Les textes sont stockés dans des structures dédiées et sont facilement extensibles si vous souhaitez ajouter de nouvelles langues.

Le support multilingue s'applique à toutes les méthodes de la classe Date (getMonthName(), getDayName(), toString()) et de la classe Calendar (printMonth(), printYear()). Les formats d'affichage peuvent également être personnalisés en utilisant les chaînes de format ("DD/MM/YYYY", "MONTH DD, YYYY", "DAY, DD MONTH YYYY", etc.).

Fêtes et jours spéciaux — Intégration culturelle

Calendar++ inclut une base de données de fêtes religieuses et civiles pour plusieurs calendriers. Vous pouvez obtenir les fêtes d'une date donnée avec la méthode CalendarUtils::getHoliday() ou afficher les jours importants d'un calendrier spécifique.

Les fêtes couvertes incluent les grandes célébrations internationales et religieuses : Noël et Pâques (chrétiennes), l'Aïd al-Fitr et l'Aïd al-Adha (islamiques), Rosh Hashanah et Yom Kippour (juives), le Nouvel An chinois, Nowruz (persan), et bien d'autres. Cette fonctionnalité est particulièrement utile pour les applications de planification, d'agenda ou de sensibilisation culturelle.

La base de données de fêtes est extensible et peut être enrichie selon les besoins de votre application. Les fêtes sont organisées par calendrier et par date, ce qui permet une recherche efficace et une intégration transparente dans vos fonctionnalités.

Validation et normalisation — Robustesse assurée

Calendar++ offre des mécanismes de validation et de normalisation des dates pour garantir la robustesse de vos applications. La méthode isValid() permet de vérifier si une date est valide dans son calendrier, tandis que normalize() permet de corriger automatiquement une date invalide.

La validation est essentielle pour traiter les entrées utilisateur : elle permet de détecter les dates impossibles comme le 31 février ou le 30 février, ou les années négatives dans certains calendriers. La normalisation, quant à elle, convertit une date invalide en une date valide en ajustant les composantes (par exemple, 31 février devient 28 ou 29 février selon l'année).

Ces mécanismes sont intégrés directement dans le constructeur de la classe Date : si une date invalide est créée, elle est automatiquement normalisée en 1er janvier 2000. Vous pouvez également appeler explicitement ces méthodes pour un contrôle plus fin de vos données.
